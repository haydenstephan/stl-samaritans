# St. Louis Samaritans Website Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build and deploy a five-page warm/personal static website for the St. Louis Samaritans homeless ministry, hosted on Hayden's personal GitHub Pages at `stlsamaritans.org`.

**Architecture:** Astro static site generator with Tailwind CSS, deployed via GitHub Actions to GitHub Pages. Forms via Formspree, events via embedded Google Calendar, donations via Venmo deep links. No backend, no database, no auth.

**Tech Stack:** Astro 5+, Tailwind CSS, TypeScript, GitHub Pages, GitHub Actions, Formspree, Google Calendar embed.

**Verification model:** This is a static marketing site — `astro check` (type checking) and `astro build` (catches broken imports, missing components, dead links) are the primary integration tests. Visual verification happens via `astro dev` after each page is built. There is no traditional unit-test suite; the build is the test.

- [ ] **Step 1: Create `src/components/FormWrapper.astro`**

```astro
---
import { FORMS } from '../config';

interface Props {
  subject: string;
  successMessage?: string;
}
const { subject, successMessage = "Thank you. We'll be in touch soon." } = Astro.props;
---

<form
  action={FORMS.endpoint()}
  method="POST"
  class="space-y-4 max-w-lg"
  data-form-wrapper
>
  <input type="hidden" name="_subject" value={subject} />
  <slot />
  <button
    type="submit"
    class="bg-clay text-cream px-6 py-3 font-serif tracking-wide uppercase text-sm hover:bg-ink transition-colors"
  >
    Send
  </button>
</form>

<div data-form-success class="hidden bg-sand border border-warmth p-6 max-w-lg">
  <p class="text-clay font-serif text-lg">{successMessage}</p>
</div>

<script>
  document.querySelectorAll('[data-form-wrapper]').forEach((form) => {
    form.addEventListener('submit', async (e) => {
      e.preventDefault();
      const f = e.target as HTMLFormElement;
      const data = new FormData(f);
      try {
        const res = await fetch(f.action, {
          method: 'POST',
          body: data,
          headers: { Accept: 'application/json' },
        });
        if (res.ok) {
          f.classList.add('hidden');
          const success = f.parentElement?.querySelector('[data-form-success]');
          success?.classList.remove('hidden');
        } else {
          alert('Something went wrong. Please email us directly.');
        }
      } catch {
        alert('Something went wrong. Please email us directly.');
      }
    });
  });
</script>
```

The `_subject` hidden field is a Formspree convention — it sets the email subject of the notification.

- [ ] **Step 3: Create `.github/workflows/deploy.yml`**

```yaml
name: Deploy to GitHub Pages

on:
  push:
    branches: [main]
  workflow_dispatch:

permissions:
  contents: read
  pages: write
  id-token: write

concurrency:
  group: pages
  cancel-in-progress: false

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: npm
      - run: npm ci
      - run: npm run build
      - uses: actions/upload-pages-artifact@v3
        with:
          path: ./dist

  deploy:
    needs: build
    runs-on: ubuntu-latest
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    steps:
      - id: deployment
        uses: actions/deploy-pages@v4
```

This is the modern GitHub Pages flow — no `gh-pages` branch, no PAT needed, uses the built-in `GITHUB_TOKEN`.
